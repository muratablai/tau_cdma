    ┌────────────────────────────┬────────────────────────────┬──────────────┐
    │ CDMA Receiver               │ Physics Equivalent         │ Bound        │
    ├────────────────────────────┼────────────────────────────┼──────────────┤
    │ Matched filter              │ 1D histogram fit           │ CRB(1D)      │
    │ Decorrelating detector      │ R⁻¹ template fit           │ CRB(1D)      │
    │ MMSE linear receiver        │ Regularized template fit   │ ≤ CRB(1D)    │
    │ NN single-user detector     │ Mass-only NN classifier    │ ~ CRB(1D)    │
    │ NN multiuser receiver       │ Mass+features NN           │ → CRB(m+PID) │

    └────────────────────────────┴────────────────────────────┴──────────────┘

    Qualitative interpretation:
      The histogram template fit is a MATCHED FILTER: it correlates
      the observed spectrum with each template (spreading code) and
      extracts branching ratios. It suffers from MAI — channels with
      overlapping hadronic templates interfere, inflating estimation variance.

      Adding detector features (PID, tracks, π⁰ count) is equivalent
      to WIDEBAND CDMA: more spreading dimensions resolve previously
      aliased users. The multi-feature NN acts as a nonlinear MMSE
      receiver that approaches the multi-dimensional CRB.

      INFERNO/IMNN methods explicitly train NNs to maximize Fisher
      information — they are FISHER-OPTIMAL SUMMARY CONSTRUCTORS.
      In nuisance-free settings, this aligns with Bayes-optimal
      classification. General optimality is not proven.
      The CRB ratio provides a heuristic scale for achievable gain:
        gain_k ~ σ_1D(k) / σ_5D(k) = √[CRB_1D / CRB_5D]

      The NN does NOT beat the CRB. What improves is WHICH CRB:
      going from 1D to multi-D lowers the bound because the FIM gains
      rank (aliased eigenvalues become non-degenerate). The NN is the
      practical tool to approach this tighter bound.
